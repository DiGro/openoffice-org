Openoffice.org 2.3.x og SELinux

OpenOffice 2.3.x er i nogle tilfælde inkompatibel med SELinux i enforcing mode. Installationen forløber normalt, men programmet kan ikke starte. Der arbejdes på en løsning, som formentlig kommer i version 2.4.0.

Hvis kommandoen:

setenforce 0

som root løser problemet (midlertidig deaktivering af SELinux, se også http://www.openoffice.org/issues/show_bug.cgi?id=80816), anbefales et af følgende workarounds:

Første løsningsforslag:

Åbn en shell som root

Kør denne kommando (evt. skal stien være en anden):

chcon -t textrel_shlib_t /opt/openoffice.org2.3/program/libvclplug_gen680li.so.1.1

Andet løsningsforslag:

Kør

system-config-securitylevel

eller åbn Konfiguration af sikkerhedsniveau eller hvad din distribution bruger

Vælg fanen SELinux
Vælg Ændr
SELinux-retningslinjer
Under Compatibility sættes flueben ud for "Allow the use of shared libraries with Text Relocation"
